## ⚠️ 审查核心标准 (The 10k Users Standard)

### 1. 业务一致性验收 (PRD Alignment) - **最重要！**
* **功能完整性**：代码是否**完整实现**了 PRD 描述的所有逻辑？(例如：若 PRD 要求“区域优先排序”，代码里有没有体现？)
* **业务边界**：是否越界做了不该做的功能（如在线修图、支付）？或者漏做了关键约束（如审核状态默认值）？
* **字段准确性**：JSON 字段名、枚举值（Enum）是否与 PRD 及模型定义 **100% 一致**？

### 2. 高并发与性能 (Performance & Scalability)
* **1万用户承载力**：
    * **查询优化**：所有用于筛选 (`where`) 和排序 (`orderBy`) 的字段，是否在 JSON 模型中标记了 `"x-filter": true` 或 `"x-sort": true`？(否则会全表扫描导致卡死)。
    * **分页强制**：是否强制限制了 `limit` 最大值（防止一次拉取1000条数据撑爆内存）？
    * **避免 N+1**：是否存在循环调用数据库的情况？

### 3. 系统稳定性 (Stability & Protocol)
* **底层协议红线**：是否严格使用了 `tcb/client.go` 的通用 `Filter` 模式？(严禁硬编码 `$eq`，严禁使用 mongo-driver)。
* **容错处理**：
    * 错误返回是否脱敏？(不能把 `database timeout` 原样抛给前端)。
    * 是否处理了空指针风险？
* **安全防篡改**：Update 接口是否剥离了 `_id`, `_openid`, `audit_status` 等敏感系统字段？
